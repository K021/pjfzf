## 1. 요구사항

- macOS
- zsh (기본 셸)
- [fzf](https://github.com/junegunn/fzf)

fzf가 없으면:

```bash
brew install fzf
```

## 2. 설치

### 방법 A: 직접 source (권장)

이 저장소를 클론하고 `.zshrc`에 한 줄 추가:

```bash
# 저장소 위치에 맞게 경로 수정
echo 'source ~/projects/pjfzf/pj.zsh' >> ~/.zshrc
source ~/.zshrc
```

### 방법 B: Homebrew (릴리스 후)

```bash
brew tap kimjooeon/pjfzf
brew install pjfzf
# 설치 후 안내에 따라 .zshrc에 source 추가
```

### 설치 확인

```bash
pj help
```

아래 출력이 나오면 성공:

```
pj - Project directory navigator

Usage:
  pj              Select project interactively with fzf
  pj <query>      Select project with initial search query
  pj add <path>   Add a base directory
  pj remove <path> Remove a base directory
  pj list         Show registered base directories
  pj help         Show this help message
```

## 3. 초기 설정

첫 실행 시 자동으로 생성되는 것들:

| 파일 | 경로 | 설명 |
|------|------|------|
| 설정 | `~/.config/pj/config` | base directory 목록 |
| 히스토리 | `~/.config/pj/history` | 접근 기록 |

기본 base directory는 `~/projects`이다. 다른 폴더를 쓴다면:

```bash
# ~/work 도 추가
pj add ~/work

# ~/projects 가 필요 없다면 제거
pj remove ~/projects
```

## 4. 기본 사용법

### 인터랙티브 선택

```bash
pj
```

fzf 팝업이 뜨고, 화살표로 선택하거나 타이핑해서 검색:

```
pj> _
> /Users/you/projects/my-app
  /Users/you/projects/api-server
  /Users/you/work/dashboard
  4/4
```

Enter로 선택하면 해당 디렉토리로 이동한다.
ESC를 누르면 아무 일도 일어나지 않는다.

### 검색어와 함께

```bash
pj api
```

"api"가 초기 검색어로 들어간다. 매치가 하나뿐이면 자동으로 바로 이동한다.

### 실전 예시

```bash
# "sync" 가 들어간 프로젝트로 바로 이동
pj sync

# fuzzy matching으로 "db"만 쳐도 "ipd-database-sync" 찾기 가능
pj db
```

## 5. base directory 관리

### 추가

```bash
pj add ~/work
pj add ~/personal
```

- 실제 존재하는 디렉토리만 추가 가능
- 중복 추가 시 경고

### 제거

```bash
pj remove ~/work
```

### 목록 확인

```bash
pj list
```

출력 예시:

```
Base directories:
  /Users/you/projects
  /Users/you/work
```

## 6. 정렬 원리 (frecency)

프로젝트 목록은 **frecency** (frequency + recency) 점수로 정렬된다.

```
점수 = 접근 횟수 x 시간 가중치
```

| 마지막 접근 | 가중치 |
|------------|--------|
| 1시간 이내 | x16 |
| 1일 이내 | x8 |
| 1주 이내 | x4 |
| 1달 이내 | x2 |
| 그 외 | x1 |

**예시**: 오늘 5번 접근한 프로젝트 = 5 x 16 = 80점, 한 달 전 20번 접근한 프로젝트 = 20 x 1 = 20점 → 오늘 자주 쓴 프로젝트가 위에 온다.

한 번도 접근하지 않은 프로젝트는 히스토리 항목 뒤에 알파벳순으로 표시된다.

## 7. 설정 파일 직접 편집

`pj add`/`pj remove` 대신 직접 편집해도 된다:

```bash
vim ~/.config/pj/config
```

```
# base directories (한 줄에 하나)
~/projects
~/work
~/personal
```

- `#`으로 시작하는 줄은 주석
- 빈 줄은 무시
- `~`는 자동으로 홈 디렉토리로 확장

히스토리 초기화가 필요하면:

```bash
> ~/.config/pj/history
```

## 8. 문제 해결

### "pj: fzf is required"

```bash
brew install fzf
```

### "pj: command not found"

`.zshrc`에 source 라인이 있는지 확인:

```bash
grep pj.zsh ~/.zshrc
```

없으면 추가:

```bash
echo 'source /path/to/pj.zsh' >> ~/.zshrc
source ~/.zshrc
```

### fzf 목록이 비어 있다

base directory가 올바른지 확인:

```bash
pj list
ls ~/projects  # 하위 디렉토리가 있는지 확인
```

### Tab 완성이 안 된다

zsh completion system이 초기화되어 있어야 한다. `.zshrc`에 아래가 있는지 확인:

```bash
autoload -Uz compinit && compinit
```

`source pj.zsh`보다 **위에** 있어야 한다.
