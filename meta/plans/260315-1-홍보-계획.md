## 1단계: 사전 준비

> 목표: 홍보에 필요한 자료를 완성한다.

### 1-1. 터미널 데모 영상 제작

- **도구**: [asciinema](https://asciinema.org) 또는 [vhs](https://github.com/charmbracelet/vhs) (GIF 변환 가능)
- **시나리오**:
  1. `pj` 실행 → fzf로 프로젝트 선택 → 이동
  2. `pj add ~/work` → 베이스 디렉토리 추가
  3. `pjmk new-app` → 프로젝트 생성
  4. Tab 완성 데모
- **길이**: 30초 이내
- **산출물**: GIF 파일 (README에 삽입), asciinema 링크

### 1-2. README 보강

- [ ] 스크린샷을 GIF 데모로 교체 (또는 병행)
- [ ] README.md를 영문으로 전환 (GitHub 기본 노출용), 한국어 버전은 README.ko.md로 분리
- [ ] 유사 도구와의 차별점 명시 (z, zoxide, autojump 대비)
  - **프로젝트 디렉토리 중심**: 임의 경로가 아닌, 등록된 베이스 디렉토리 하위의 프로젝트에 집중 → 노이즈 없이 프로젝트만 탐색
  - **fzf 네이티브 통합**: 프리뷰 창으로 디렉토리 내용을 즉시 확인 (zoxide는 자체 매칭 알고리즘 사용)
  - **`pjmk`로 프로젝트 생성**: 네비게이션뿐 아니라 프로젝트 디렉토리 생성까지 지원
  - **Tab 완성 통합**: `pj ` + Tab으로 fzf 실행

### 1-3. 게시글 템플릿 작성

- 제목: 간결하고 흥미를 끄는 형태 (예: "pjfzf – a frecency-based project directory navigator for zsh")
- 본문 구성: 문제 → 해결 → 데모 → 설치 방법 → 피드백 요청
- **Reddit 주의사항**: 셀프 프로모션 비율 규칙 준수 (계정 활동의 10% 이하를 자기 홍보로 유지). 게시 전에 해당 서브레딧에서 일정 기간 활동(댓글, 다른 글 참여)하는 것이 좋음.

## 2단계: 1차 홍보 — zsh 커뮤니티

> 목표: 정확한 타겟 사용자에게 먼저 노출하고 피드백을 받는다.

| 채널 | 행동 | 기대 효과 |
|------|------|-----------|
| **r/zsh** | 게시글 작성 + GIF 데모 | 핵심 사용자 피드백 확보 |
| **awesome-zsh-plugins** | PR 제출 | 검색 유입 + 신뢰도 |

### 타이밍

- r/zsh 게시 후 1~2주간 피드백 수집
- 버그 수정 및 기능 개선 반영

## 3단계: 2차 홍보 — 범용 CLI 커뮤니티

> 목표: CLI 도구에 관심 있는 더 넓은 사용자층에 노출한다.

| 채널 | 행동 | 비고 |
|------|------|------|
| **r/commandline** | 게시글 작성 | CLI 도구 전문 서브레딧, 규모 큼 |
| **r/linux** | 게시글 작성 | Linux 사용자 대상 |

### 타이밍

- 1차 홍보 피드백 반영 완료 후 진행
- 각 채널에 1~2일 간격으로 올려서 동시다발적 노출 방지

## 4단계: 3차 홍보 — 대형 채널

> 목표: 바이럴 가능성이 있는 대형 채널에 도전한다.

| 채널 | 형식 | 비고 |
|------|------|------|
| **Hacker News** | "Show HN: pjfzf – ..." | 타이밍과 제목이 중요, 미국 시간대 오전 게시 |
| **Dev.to / Hashnode** | 블로그 글 | 제작 과정 + 사용법 튜토리얼 형태 |

### 사전 조건

- GitHub 스타 최소 10~20개 확보 (신뢰도)
- README와 데모가 완성도 높은 상태

---

## 5단계: 지속 관리

> 목표: 일회성 홍보가 아닌 지속적 성장을 만든다.

- **피드백 대응**: Issue/PR에 빠르게 응답 (24시간 이내)
- **변경 로그 관리**: 릴리스마다 CHANGELOG 작성
- **기능 확장 고려** (피드백 기반):
  - bash 지원 추가 → 사용자층 확대
  - fish 지원 → 완전한 크로스 셸
- **버전 릴리스**: 의미 있는 개선마다 태그 릴리스 + GitHub Release
