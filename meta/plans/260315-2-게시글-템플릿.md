## 1. r/zsh 게시글

**제목**: pjfzf – a fuzzy project directory navigator for zsh

**본문**:

I built a small zsh plugin to solve a problem I kept running into: jumping between project directories scattered across multiple base paths.

**The problem**: Tools like `z` and `zoxide` track every directory you visit. When you have dozens of projects, the results get noisy — you end up jumping to `node_modules` or random subdirectories.

**The solution**: `pjfzf` only navigates registered project directories. You register base paths (like `~/projects`, `~/work`), and it lists their immediate subdirectories with fzf. Frecency sorting puts your most-used projects first, and the fzf preview shows directory contents before you jump.

It also includes `pjmk` to create new project directories, and Tab completion that launches fzf inline.

**Demo**: [GIF or asciinema link]

**Install**:
```
brew tap K021/pjfzf && brew install pjfzf
```
Or: `curl -o ~/.pj.zsh https://raw.githubusercontent.com/K021/pjfzf/main/pj.zsh`

GitHub: https://github.com/K021/pjfzf

Would love feedback — especially on the frecency algorithm and any edge cases I might be missing.

## 2. r/commandline 게시글

**제목**: pjfzf – fzf-powered project directory navigator (zsh)

**본문**:

Built a small CLI tool for developers who juggle multiple projects across different directories.

Instead of tracking every `cd` like z/zoxide/autojump, `pjfzf` focuses only on your project directories. Register your base paths, and it gives you a clean fzf list with preview and frecency sorting.

Features:
- `pj` — fuzzy-find and jump to projects
- `pjmk` — create new project directories
- Tab completion with inline fzf
- Frecency-based sorting (frequency × recency)

**Demo**: [GIF]

Requires zsh + fzf. Install: `brew tap K021/pjfzf && brew install pjfzf`

GitHub: https://github.com/K021/pjfzf
