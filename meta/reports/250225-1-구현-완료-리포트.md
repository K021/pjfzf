## 구현 요약

단일 파일 `pj.zsh` (~160줄)로 프로젝트 디렉토리 네비게이터를 구현했다. 핵심 기능:
- base directory 아래 1-depth 프로젝트 디렉토리 수집
- frecency(빈도+최근성) 기반 정렬
- fzf 인터랙티브 UI로 선택 후 `cd`
- config/history 자동 관리

## 구현된 기능

### 내부 함수

| 함수 | 역할 |
|------|------|
| `_pj_init()` | `~/.config/pj/` 디렉토리 + config + history 자동 생성 |
| `_pj_list()` | config의 base_dirs에서 1-depth 디렉토리 수집 |
| `_pj_score()` | frecency 점수 계산 (접근횟수 × 시간가중치) |
| `_pj_sorted()` | frecency 내림차순 + 미접근 항목 알파벳순 정렬 |
| `_pj_update()` | 히스토리 파일 업데이트 (접근시각, 횟수) |
| `_pj_add_base()` | base directory 추가 (중복/존재 검증) |
| `_pj_remove_base()` | base directory 제거 |
| `_pj_show_config()` | 등록된 base directories 표시 |
| `_pj_help()` | 사용법 출력 |
| `_pj()` | zsh Tab 완성 함수 (`compdef` 연동) |

### CLI 서브커맨드

| 커맨드 | 동작 |
|--------|------|
| `pj` | fzf 인터랙티브 선택 → cd |
| `pj <query>` | 초기 검색어로 fzf 실행 |
| `pj add <path>` | base directory 추가 |
| `pj remove <path>` | base directory 제거 |
| `pj list` | base directories 표시 |
| `pj help` | 도움말 |

### Frecency 가중치

| 최근 접근 시간 | 가중치 |
|---------------|--------|
| 1시간 이내 | ×16 |
| 1일 이내 | ×8 |
| 1주일 이내 | ×4 |
| 1달 이내 | ×2 |
| 그 외 | ×1 |

### Edge case 처리

- config/history 파일 미존재 시 자동 생성 (기본값: `~/projects`)
- 존재하지 않는 base directory는 리스팅에서 제외
- 히스토리의 삭제된 프로젝트(stale entry)는 정렬 목록에서 제외
- fzf 미설치 시 안내 메시지 출력
- `compdef` 미로드 환경에서도 안전하게 동작

### 테스트 커버리지

- `_pj_init`: 디렉토리/파일 생성, 기본 config 내용
- `_pj_list`: 단일/복수 base_dir, 주석/빈줄 무시
- `_pj_score`: 5개 시간 구간별 가중치 검증
- `_pj_update`: 신규 추가, 횟수 증가, 복수 항목
- `_pj_sorted`: frecency 정렬 순서, 미접근 항목 알파벳순
- `_pj_add_base`: 정상 추가, 중복 거부, 미존재 경로 거부
- `_pj_remove_base`: 정상 제거, 미존재 항목 거부
- `_pj_show_config`: 출력 내용 검증
- `_pj_help`: 출력 내용 검증
- `pj` 메인 함수: list/help/add/remove 서브커맨드
- Edge case: 미존재 base_dir, stale history entry
